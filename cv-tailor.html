User flow

From a typed keyword to a queued application, in five hops.

I press Start in the dashboard. A controller subprocess fans out scrapers, an analyst worker, and a score worker. Each stage writes a small JSON artefact to disk so the next stage can pick it up independently. The whole thing is restart safe.

⚲

Discover

Playwright scrapes Naukri and LinkedIn with stealth flags so Akamai does not flag the session. Listings dedupe across runs via a stable hash.

∿

Enrich

Each card gets a JD body fetch, location and work mode parse, post date normalise, and an Apollo plus Hunter contact resolver for the hiring manager.

⌬

Score

Six component ATS scorer (TF IDF, semantic via MiniLM, skills, seniority, years of experience, role match) writes a row to the day's xlsx and an inbox task for Claude.

✦

Tailor

Headless Claude reads the JD, scores against master_cv.json, and writes master_fit, tailoring_plan, tailored_cv_delta and tailored_fit. Cover letter too if a real recruiter email exists.

◈

Render

Three integrators run in parallel. Each renders a tailored PDF with fpdf2, drafts the .eml with the PDF attached, and updates the xlsx row.

⛨

Approve

Nothing ships until I tick the per row Approve box AND set CV_AGENT_ENABLE_AUTO_SUBMIT=true. Two keys. Belt and braces.

Architecture

A queue between every stage. A guardrail at every model call.

The system is a stack of small workers, each pulling from a filesystem queue and writing artefacts the next worker can read. It runs entirely on my laptop and is observable from the Next.js dashboard at every step.

Safety & correctness

Five layers between the model and the recruiter's inbox.

A senior PM CV cannot be a hallucination. Every model output passes through evidence checks before it touches the PDF, and nothing leaves the laptop without two explicit keys.

①Evidence only

Skills, metrics, employers and tools must already exist in master_cv.json. Anything else gets dropped by a post filter, not asked again.

②Banned phrase scrubber

Generic LLM tells like "data driven decisions" or "results driven leader" never reach the PDF. The list grows whenever I spot a new tell.

③Numeric guard

Every percentage, count and currency figure must round trip back to a number that exists in the master CV. No invented metrics.

④Two key submit gate

An application ships only when the per row Approve checkbox is on AND the global env flag is true. Either alone is a no op.

⑤384+ assertion tests

Hard guarantees. No submits in dry run. No live email sends in tests. No full secrets logged. CI fails loudly.

⑥JD archive trail

Every JD body, every score, every tailored output is preserved in dated HTML and JSONL. Any decision can be traced back to its evidence.

Local LLM training

The Claude runs are also a labelled gold set for the local model.

Each Claude analysis is appended to a curated gold set under data/gold/. The local Mistral and Qwen analysts then learn from those examples, and once their scores agree with Claude's on a sample, the local model can take over. The whole loop runs on my laptop.

Gold set as the training signal

Every Claude run yields a labelled triple of JD plus master CV plus tailored output. That is the training data.

  • Append on success, never on mismatch noise.
  • Top k gold cases retrieved per new JD via MiniLM embeddings.
  • Few shot prompt for the local LLM is rebuilt from the freshest cases.
  • A quality gate compares the local output to Claude's score on a held out sample before promoting.
  • When the gate stays green for N runs, the local LLM owns the path. Claude becomes the audit trail.

What I am tuning for

PM voice, Indian English register, evidence first sentences, and tight bullet rewrites that do not invent.

  • Headline that echoes the JD's seniority, never promotes beyond it.
  • Summary opener that maps my closest analog to the JD's first concern.
  • Skills reordered around the JD top keywords, drawn only from CV evidence.
  • Bullet priority per company, not bullet rewrite, to keep numbers honest.
  • Cover letter open that quotes a concrete metric from CV evidence.
